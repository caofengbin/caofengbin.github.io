ButterKnife的使用与原理(1)

1.概述

  ButterKnife是一个依托 Java 的注解机制来实现辅助代码生成的框架。对于注解机制还不是太熟悉或者了解的同学，可以参见笔者自己总结的一篇文章–Java注解机制基础。本文大致讲述ButterKnife常用的特性和使用过程中的一些要点。并分别通过反射机制以及apt(Annotation Processing Tool)模拟实现ButterKnife的效果。希望可以让大家了解到 Java 注解机制的强大之处，并对 ButterKnife，Dagger 这样类似的框架有一定的认识。

2.ButtrerKnife的简介与使用



  作为一名Android开发，我们总是会写大量的findViewById和点击事件，像初始view、设置view监听这样简单而重复的操作，当然不会偷懒的程序员不是好程序员，自然也出现了相应的解决方案–依赖注入。而ButterKnife则是依赖注入中相对简单易懂的一个开源框架，让开发人员从这些烦人臃肿的代码中解脱出来。ButtrerKnife目前最新的版本已经更新到了8.4.0，本文的内容也是基于最新版本进行。

3.使用ButtrerKnife的相关注意事项

(1)ButterKnife的优势

通过上文的描述使用，我们可以发现ButterKnife的优势主要体现在如下几点：

  • (1)强大的View及资源绑定，以及多个事件处理功能，简化代码，提升开发效率；
  • (2)方便的处理Adapter里的ViewHolder绑定问题；
  • (3)使用apt技术在编译时生成代码，运行时不会影响应用的效率；
  • (4)增加的代码的清晰度和可读性；

(2)ButterKnife使用的几个注意点

  • (1)在Activity中注入View时，ButterKnife.bind(this)的调用必须在setContentView();之后，且父类bind之后，子类不需要再bind；
  • (2)在Fragment中注入View时，需要在onCreateView()方法中，使用ButterKnife.bind(this, mRootView);
  • (3)在Fragmet生命周期中，onDestroy()方法中需要调用unbind()方法；
  • (4)setContentView不能通过注解实现，ButterKnife中没有提供相应注解；
  • (5)对需要注入的成员，不能用private或者static修饰，否则会报错(后文原理部分会详细的分析原因)；
  • (6)ButterKnife7.0中一个比较大的API变化：
    在7.0以前的版本中，叫@InjectView，在7.0版本中，叫@Bind，在8.0版本中，改成了@BindView；
    在7.0以前的版本中使用ButterKnife.inject(this); 来进行注入
    在7.0以后的版本中使用ButterKnife.bind(this); 来进行注入
  • (7)ButterKnife不能在项目中的library module中使用，这是因为library中的R字段的id值不是final类型的，但是我们的应用module却是final类型的。

4.使用反射模拟实现ButterKnife的效果

  ButterKnife是一个依托Java的注解机制来实现辅助代码生成的框架。关键之处在于如何处理这些自定义的注解来生成代码。根据自定义注解时指定的Retention值不同，注解的解析方式主要分为两种：运行时解析和编译时解析。在本节，我们主要采用运行时解析的方式，来模拟实现ButterKnife的效果，加深对这个框架及java注解机制的理解。

(1)反射相关API

  运行时解析注解利用的核心技术就是java反射，核心接口为：AnnotatedElement，引用一下官方文档对他的解释：

  AnnotatedElement接口表示包含注解的元素。java.lang包中的Class和Package，以及java.lang.reflect包中的Constructer,Field和Method都实现了AnnotatedElement接口。
AnnotatedElement接口中常用的几个方法：

  • (1) getAnnotations()：
    Returns all annotations present on this element;
  • (2) getAnnotation(Class annotationClass)：
    Returns this element’s annotation for the specified type if such an annotation is present, else null.
  • (3) getDeclaredAnnotations()：
    Returns all annotations that are directly present on this element.

更多详细信息可以参见Java处理注解相关的反射API

  到这里我们就通过反射机制模拟实现了ButterKnife的效果了。这种方式虽然简单，但是反射多多少少会有性能上的损耗。ButterKnife最新版本也不是通过反射来完成的，而是利用APT(Annotation Processing Tool)编译时解析技术，动态生成绑定事件或者View的java代码，然后在运行的时候，直接调用bind方法完成绑定。在下一篇文章中，会详细分析apt技术的关键点，并通过apt模拟实现ButterKnife的效果。
